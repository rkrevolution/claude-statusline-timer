# LinkedIn post (primary)

If you're like me, you've gotten a lot of value out of Claude Code. But new tools mean new processes.

Consider this: where did the time go?

I wanted to know. Across every session. Across multi-agent workflows. Across git worktrees and parallel terminals. How much of my day was actually spent with Claude?

Well, here's the ticket. That's what I built.

A status bar that sits quietly in the terminal and keeps tabs on every session across the day. It correctly handles parallel work (no double-counting when you're running three agents at once). It trims idle time (walking away doesn't inflate your total). And it shows a live countdown on the 5-hour rate limit so you're never surprised by it again.

Runs 100% local. Zero API tokens consumed. Built on Claude Code's official status line hook.

If you're running Claude Code heavy and want to know where the hours are actually going:

https://github.com/rkrevolution/claude-statusline-timer
